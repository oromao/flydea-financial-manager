# GEMINI.md

## Antigravity behavior
- Neste workspace, trate o Obsidian brain como memória externa primária.
- Sempre use o fluxo: context -> CAG -> RAG -> execução -> write-back.
- Não trabalhar só por contexto de sessão quando houver conhecimento no brain.
- Para tarefas complexas, gerar plano curto antes da execução.
- Sempre informar quais notas ou grupos do brain foram usados.

## Quando acionar brain workflow
Acione esse fluxo quando o pedido mencionar:
- brain
- obsidian
- contexto anterior
- projeto atual
- arquitetura
- decisões
- documentação
- runbook
- rag
- cag
